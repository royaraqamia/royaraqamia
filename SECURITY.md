# Security Policy

## Private Codebase

This is a **private, confidential codebase**. Security vulnerabilities should be reported directly to the development team.

## Reporting Security Vulnerabilities

Do not create public issues for security concerns. Report them directly to:

- **Email**: contact@royaraqamia.com
- **Response Time**: We aim to acknowledge reports within 48 hours and provide a status update within 5 business days.

## Disclosure Policy

We follow a coordinated disclosure process:

1. Reporter submits vulnerability via email
2. We acknowledge receipt within 48 hours
3. We investigate and develop a fix
4. Fix is deployed to production
5. Public disclosure (if any) is coordinated with the reporter

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Scope

This policy applies to the production website and its source code. The following are considered in scope:

- Cross-site scripting (XSS)
- Cross-site request forgery (CSRF)
- Remote code execution (RCE)
- Authentication/authorization bypass
- Sensitive data exposure

## Confidentiality

All code, configurations, and documentation in this repository are confidential and proprietary.
